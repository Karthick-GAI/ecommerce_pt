Architecture Decision Records — 12 ADRs total
Six decisions that shaped the system
ADR-001
pgvector — not Pinecone
Single SQL query runs vector cosine search AND price/category filter atomically. Dedicated vector DBs need two network hops.
Trade-off: at 500K+ products Pinecone wins on ANN performance. Abstraction layer (vector_store.py) makes migration a one-file change.
ADR-003
GPT-5.4-mini — not GPT-4o
5× cheaper. P95 latency 1.8s vs 3.5s (would violate 3s NFR). Structured JSON extraction is reliable without brittle post-processing.
Trade-off: marginal quality gap. Mitigated by circuit breaker + Flan-T5 local fallback.
ADR-004
text-embedding-3-small — not ada-002
6.5× cheaper than ada-002, higher MTEB score (62.3% vs 61.0%). 5,000 product embeddings cost ~$0.01.
Trade-off: not the highest-quality model. 2.3% MTEB gap vs 3-large doesn't justify 6.5× cost at this scale.
ADR-005
Sync HTTP — not Kafka
All primary user journeys (search, checkout) are request-response. The UI blocks waiting for a result — async messaging adds latency without reducing it.
Trade-off: no at-least-once delivery. Production fix: payment confirmation webhook moves to Kafka for decoupling.
ADR-011
LLM-as-Judge — not regex guardrails
Handles rephrasing and novel harmful patterns that static wordlists miss. Scores 5 dimensions: relevance, grounding, completeness, helpfulness, conciseness.
Trade-off: +40ms per request. Acceptable within 3s P95 NFR. Falls back to keyword filter if Azure is unavailable.
ADR-012
Resilience across all 5 Azure-dependent services
shopping_assistant: Flan-T5-base → keyword ILIKE. product_catalogue, multi_agent router/tools, tool_calling_agent: timeout + graceful keyword/message fallback. checkout: db.rollback() on commit failure. Every service flags its failure mode.
Trade-off: Flan-T5 quality 3.2/5.0 vs GPT's 4.34/5.0. AZURE_TIMEOUT_SECS is env-configurable per service. System recovers automatically when Azure returns. See Slide 7 for full coverage table.

Summary
What worked, what to improve, what comes next

What worked well

pgvector composability — SQL + ANN in one query is the single biggest accuracy win (+4pp NDCG vs semantic-only)
Hallucination guard — 4.7/5.0 grounding score; GPT faithfully reports retrieved prices and specs
Graceful degradation — two-tier fallback means zero 503s even when Azure is unavailable
Purchase history personalisation — measurably different, warmer responses with user_id present

Known limitations

Books Recall@5 = 0.58 (below 0.60) — descriptions lack structured metadata (author, ISBN)
P99 RAG latency 3.1s — above soft target; fix: pre-warm embedding at startup + response streaming
Cold-start recommendations score 2.9/5 helpfulness — trending fallback too generic without history
In-process rate limit counters don't aggregate across replicas — needs Redis in production

Production path

Migrate to K8s (EKS) + Istio service mesh; HPA on CPU/RPS
Swap pgvector → Qdrant at 500K+ products (one-file change by design)
Redis-backed circuit breakers, rate limits, session state across replicas
MLflow for embedding model versioning + ArgoCD GitOps CI/CD
