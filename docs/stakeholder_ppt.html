Why Microservices?

Independent scaling, fault isolation, team parallelism. Each service deployable independently.

Why pgvector over Pinecone?

SQL + ANN in one query — no two-phase network hop. Migration path: swap vector_store.py only.

Why sync HTTP over Kafka?

Primary flows are request-response. Async messaging adds latency without reducing it for browse/checkout.

Data Flow

Two Critical Pipelines

RAG Retrieval Pipeline (Shopping Assistant)

01User Query Input
02Input Guardrail (LLM judge)
03NL Query Parsing → structured filters
04Embed query → text-embedding-3-small
05pgvector cosine search + SQL filters
06Build product context string
07GPT-5.4-mini generate reply
08Output Guardrail + return
Active steps highlighted. Key differentiator: SQL pre-filters applied at pgvector query time — no two-pass retrieval.

Data Ingestion Pipeline (Setup)

01Synthetic data generation (Faker)
02PostgreSQL bulk insert (5K products)
03Batch embed via text-embedding-3-small
04UPDATE products SET embedding = vector
05pgvector IVFFlat index creation
06Services load index at startup (cold start)
Embeddings generated once (~$0.01 for 5K products). Re-embedding triggered only on product catalogue drift.

Architecture Decision Records

Key Design Decisions & Trade-offs

Full ADRs in DECISIONS.md

ADR-001 — Vector Store
pgvector (PostgreSQL)
Won: SQL + ANN in one query — no two-phase hop, transactional consistency.
Trade-off: Not optimal at 500K+ vectors. Migration path: swap vector_store.py only (abstraction layer).
ADR-003 — LLM Model
GPT-5.4-mini (Azure)
Won: P95 < 2.3s, reliable structured JSON, Azure SLA.
Trade-off: Vendor lock-in. Mitigated by local Flan-T5 fallback + circuit breaker.
ADR-007 — Recommendations
Hybrid CF + Content + Trending
Won: Covers cold-start users (new reg/new products) from day one.
Trade-off: Static weight thresholds. Prod: A/B test or bandit algorithm.
ADR-005 — Inter-service Comms
Sync HTTP (httpx)
Won: Lowest latency for request-response; UI blocks waiting for result.
Trade-off: No at-least-once delivery. Prod: payment webhook moves to Kafka.
ADR-010 — Accuracy vs Perf
top-8 retrieval, GPT-5.4-mini
Result: NDCG@5 = 0.851, P95 = 2.28s — within NFR targets.
Trade-off: top-20 + GPT-4o gives +1% accuracy but P95 = 3.5s (violates NFR).
ADR-011 — Guardrails
LLM-as-Judge (GPT-5.4-mini rubric)
Won: Nuanced; handles rephrasing attacks; structured JSON scoring.
Trade-off: +40ms per request. Acceptable at 3s P95 budget.

Production Scale Design

POC → Production Architecture

POC (Current)

Deployment: 13 uvicorn processes on one VM

Entry: Direct ports (8001–8013)

DB: Shared PostgreSQL + SQLite on localhost

Rate limiting: In-process slowapi (per-replica)

Circuit breaker: In-process only (state lost on restart)

Secrets: .env file

Production Target

Kubernetes (EKS): HPA auto-scale 2–10 replicas per service, Istio mTLS, canary deployments

API Gateway (Kong): TLS termination, JWT validation at edge, centralised rate limiting

Load Balancer (AWS ALB): L7 routing, health checks, sticky sessions for WebSocket

Redis Cluster: Shared circuit breaker state, rate limit counters, semantic search cache (15m TTL)

RDS per domain: Independent PostgreSQL per service domain + PgBouncer connection pooling

Secrets Manager: AWS Secrets Manager / K8s Secrets, never in .env

Observability Stack

Prometheus + Grafana (RED metrics). Fluent Bit → Elasticsearch → Kibana. Jaeger distributed tracing via Istio. PagerDuty alerts on P99 > 3s.

MLOps Layer

MLflow model registry for embedding versions and recommendation weights. Drift detection: re-embed if avg cosine similarity drops >10%. Automated NDCG gate in CI pipeline.

CI/CD for ML

GitLab CI → Docker build → ArgoCD GitOps sync to staging → automated accuracy test (NDCG@5 ≥ 0.75) → manual approval → prod promotion.

ML Resiliency

Graceful Degradation & Local Fallback

Two-Tier Fallback Strategy

Tier 0 — Primary (Azure OpenAI)

GPT-5.4-mini via Azure. Full RAG pipeline. Quality: 4.34/5.0 LLM Judge score.

Tier 1 — Local Flan-T5-base (CPU)

Activates on Azure timeout (>5s) or circuit breaker open. Loaded at startup — zero cold start per request. Quality: 3.2/5.0 (lower but functional). Response labelled with fallback_mode=true.

Tier 2 — Keyword-only Results

Activates if vector indexing fails (no embeddings). Plain ILIKE query, no LLM generation. Always available (pure Python). Latency: 34ms.

Circuit Breaker Pattern

CLOSED → Azure calls succeed
  ↓ failure_count > threshold
OPEN → All calls fail fast → Flan-T5 activated
  ↓ after timeout (30s)
HALF-OPEN → One probe call
  ↓ success → back to CLOSED
  ↓ failure → back to OPEN
Implemented in src/nfr/circuit_breaker.py — thread-safe, shared across all service instances within a process. Production: Redis-backed for multi-replica state sharing.
